Evaluación de Aplicaciones

Hay momentos en que conocer si una aplicación debe ser renovada, cambiada o construida nuevamente desde cero determina la estrategia a seguir.

¿Que se evalua?

Lenguaje de Desarrollo

Se realiza inspección estática de código fuente para determinar el(los) lenguajes de desarrollo utilizados. Sirve para determinar el grado de soporte actual y futuro ese(os) lenguajes así como sus fortalezas y debilidades.

Paradigma de Programación

Se realiza inspección estática de código fuente para determinar el paradigma utilizado. Esto apunta a conocer el grado de reutilización, extensibilidad y adaptabilidad.

Framework

Se realiza una inspección estática de código fuente para determinar cual(es) framework se utiliza en la aplicación. Puede ser uno o varios y esto ayuda a saber cual puede ser el soporte futuro para la aplicación.

Patrón de Arquitectura

Se realiza una inspección de los elementos involucrados en el funcionamiento arquitectónico de la aplicación para determinar su grado de adaptaciónsoftware.

Modularidad

Se realiza inspección funcional para detectar el grado de modularidad de la aplicación con el fin de establecer su facilidad de mantenimiento y evolución.

Bases de Datos

Se verifica el tipo de base de datos (relacional y/o no-relacional) usadas, el tipo de conexión y su uso para determinar su facilidad de mantenimiento y evolución.

Infraestructura Requerida

Se analizan los diferentes componentes necesarios para que la aplicación funcione con el fin de determinar los requerimientos reales de operación.

Aseguramiento de Calidad

Se analizan los procesos actuales utilizados para construir, mantener o evolucionar la aplicación de forma que se pueda determinar el nivel de calidad existente.
